## 📢 Reportando uma Vulnerabilidade

Se você descobriu uma vulnerabilidade de segurança no PortfolioHUB, pedimos que siga os seguintes passos:

### 1. **Não abra uma Issue pública**
Para proteger os usuários, não divulgue a vulnerabilidade publicamente até que uma correção seja implementada.

### 2. **Entre em contato por e-mail**
Envie um e-mail para: **celsokakashi@gmail.com**

### 3. **Informe os detalhes**
No e-mail, inclua:
- **Descrição clara** da vulnerabilidade
- **Passos para reproduzir** o problema
- **Versão** do projeto afetada
- **Possível impacto** da vulnerabilidade
- **Sugestão de correção** (se tiver)

### 4. **Prazo de resposta**
Você receberá uma confirmação em até **48 horas úteis**.

### 5. **Atualizações**
Mantemos você informado sobre o andamento da correção a cada **5 dias**.

## 🔐 Políticas de Acesso

### Controle de Acesso
- **Repositório público** para transparência
- **Branches protegidas** com regras de pull request
- **Revisão de código** obrigatória para alterações

### Permissões
| Função | Permissão |
|--------|-----------|
| **Owner** | Acesso total |
| **Collaborator** | Push em branches protegidas |
| **Contributor** | Pull requests apenas |
